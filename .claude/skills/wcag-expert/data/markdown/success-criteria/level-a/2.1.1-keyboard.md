# 2.1.1 Keyboard (Level A)

**Guideline**: 2.1 Keyboard Accessible  
**Principle**: 2 Operable

## Understanding

All functionality of the content is operable through a keyboard interface without requiring specific timings for individual keystrokes, except where the underlying function requires input that depends on the path of the user's movement and not just the endpoints.

## Full Description

All [functionality](https://www.w3.org/TR/WCAG22/#dfn-functionality) of the content is operable through a [keyboard interface](https://www.w3.org/TR/WCAG22/#dfn-keyboard-interface) without requiring specific timings for individual keystrokes, except where the underlying
      function requires input that depends on the path of the user's movement and not just
      the endpoints.
   

   
   Note 1This exception relates to the underlying function, not the input technique. For example,
      if using handwriting to enter text, the input technique (handwriting) requires path-dependent
      input but the underlying function (text input) does not.
   

   
   Note 2This does not forbid and should not discourage providing mouse input or other input
      methods in addition to keyboard operation.
